Project Storage - P2P Marketplace

  • Category: Full-Stack Web Application
  • Project date: December, 2025
  • Project URL: github.com/AaronDDavis/project_storage
  • Live Demo: https://project-storage-app-ybvk.onrender.com/user/login

Project Overview

Peer-to-peer storage marketplace connecting space renters with individuals needing storage. Built as a full-stack demonstration of service-oriented architecture, algorithmic resource allocation, and production deployment practices.

Key Technical Contributions

Architectural Refactor: Service-Layer Pattern

Transitioned from monolithic "Fat Model/View" to Service-Oriented Architecture. Business logic extracted into dedicated service modules, enforcing Separation of Concerns and enabling framework-agnostic testing.

Design Patterns

  • Strategy Pattern: Pluggable allocation engine supporting Best-Fit, First-Fit algorithms without modifying booking workflows.
  • State Pattern: Finite State Machines managing lifecycles with validated transitions.
  • Composite Pattern: Space → Rack → Shelf hierarchy enabling aggregate queries and capacity visualization.
  • Builder Pattern: Atomic conversion of verified requests into live Spaces (preventing phantom listings).

Algorithmic Optimization

Implemented Best-Fit with Compaction strategy using Django ORM (Subquery, OuterRef). Algorithm selects racks with smallest excess capacity to minimize fragmentation, leaving larger contiguous blocks available for future bookings.

Production Deployment

Follows 12-Factor App methodology: CI/CD on Render, environment-based configuration, automated migrations, Gunicorn + WhiteNoise serving, PostgreSQL production database.

Technologies: Django 5.0, PostgreSQL, Bootstrap 5, Gunicorn, Render (PaaS)
